Absolutely Plausible — Full Architecture Overview

All systems, sites, repos, infrastructure, and team — unified operations map

Domains & Routing

  • • Two domain families: robotfantome.com + absolutelyplausible.com
  • • All DNS managed in Cloudflare (proxied)
  • • Full (Strict) SSL + Always HTTPS + Auto HTTPS Rewrites
  • • robotfantome.com → public sites (no Access)
  • • Internal subdomains on both families → Cloudflare Pages
  • • CNAME files in each repo root for custom domain mapping

Security Model

  • • Cloudflare Access = zero-trust email-based auth
  • • Public sites: open (robotfantome.com, absolutelyplausible.com)
  • • Internal sites: Access-gated (team, plan, mech, crm)
  • • Team: Luiz + Bebeco + Samuel email allowlist
  • • Each internal subdomain = separate Access application
  • • Status: Access policies still being configured

Application Layer

  • • AP Ops (mech.robotfantome.com): v1.2.0 beta — invoicing SPA
  • • AP CRM (crm.absolutelyplausible.com): v1.0.0 — CRM data model, UI in transition
  • • Both: pure static HTML/CSS/JS, no build step
  • • Data: localStorage (offline-first) → Google Sheets sync
  • • Invoice format: AP-YYYY-MMDD-001
  • • Dashboard (wgz-dashboard): planned, not yet deployed

Documentation Sites

  • • team.robotfantome.com: Dream Team workspace (EN + pt-BR)
  • • plan.robotfantome.com: Business plan + financials + strategy
  • • absolutelyplausible.com: Public company website
  • • All use MkDocs Material with Share Tech Mono
  • • Auto-deploy via Cloudflare Pages on git push
  • • Interactive: Mermaid diagrams, task lists, checklists

Data Architecture

  • • Google Sheets = single source of truth for all business data
  • • Apps Script (Gosu) = REST API layer (GET-based for CORS)
  • • localStorage = offline-first browser cache
  • • Write path: form submit → localStorage → fire-and-forget to Sheets
  • • Read path: dashboard reads localStorage only (fully offline)
  • • CSV export available for data portability

Workspace & Automation

  • • .openclaw/workspace/ = central ops hub (AGENTS.md, SOUL.md, etc.)
  • • Hermes/OWL AI agent: local macOS, automates coding + ops
  • • Obsidian vault (glober/): AI writes, Luiz browses
  • • ops/pipeline.md: lead + client tracker (updated Sundays)
  • • ops/metrics.md: weekly scorecards
  • • content/: LinkedIn, newsletter, YouTube production queue
